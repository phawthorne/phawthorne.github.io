ROOT development builds

  • Build 2018.4.4 Release Candidate
      • Changes from previous build:
    • Better handling for multiple activities. Each now has its own activity mask. This means that the activity masks are now listed in a table like most of the other file inputs.
    • Accordingly, the output tables don't have have maskpixha anymore, they have [activityname]_ha for each of the activities.
  • Build 2018.3.26
      • Changes from previous build:
    • Further change to handling of combined factors table. Now the user can type in simple formulas using +, -, *, /, ^, and some functions. Example: pop_downstr * (mv_nit + 10.0 * mv_sed). The functions abs, sqrt, and log are applied to each element in the column, and min, max, and sum can get the corresponding summary statistics.
  • Build 2018.3.23
      • Changes from previous build:
    • Better error checking for user inputs
    • Allows numeric values in combined factors tables
  • Build 2018.3.16
      • Changes from previous build:
    • Automatically makes agreement map as shapefile
    • Bug fixes and test coverage
  • Build 2017.12.2
      • Changes from previous build:
    • Change to structure of two tables: impact potential raster table and optimization weights table
    • Can now consider more than one intervention activity.
    • Additional weighting option to provide table with specific weights for each optimization run.
    • Reworked optimization results output.
    • Reworked coding to allow (many) more runs without running out of memory
  • Build 2017.10.18.3
      • Changes from previous build:
    • Changed name of column in targets table from 'var' to 'name' to be consistent with other tables.
    • Changed name of column in objectives table from 'minmax' to 'weight'.
    • Single point optimization analysis type now functional. To use, type 'single' in the Analysis Type box, and supply numeric weights in the 'weight' column in the objectives table.
    • Can use custom SDU shapefile rather than regular grid. Put the path to the shapefile in the Spatial Decision Unit Shape field. Make sure the shapefile has a field called 'SDU_ID' with unique values.
    • Can provide custom suffix to distinguish output folders for separate optimization runs.
  • Release 1.0.0
